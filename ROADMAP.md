# Roadmap

## Feito

- Base Expo com rotas principais do módulo cidadão.
- Fluxo de aceite LGPD.
- Login/cadastro com API.
- Serviços locais para autenticação, alertas, praias, localização, saúde da API, Kids e sincronização.
- Fila offline para operações críticas.
- Área Kids em modo protegido.
- Diagnóstico operacional na tela Conta.
- Publicação inicial no GitHub.
- Documentação de autoria, contribuição, checklist e setup.
- Integração com contratos Kids da API Mbeach 1.3.
- Política remota de foto infantil tipada no app.
- Registro remoto de aceite LGPD para usuário autenticado.

## Em validação

- UX dos fluxos principais em dispositivos reais.
- Mensagens de erro e estados vazios.
- Troca de ambiente de API pela tela Conta.
- Health checks `GET /health/live` e `GET /health/ready`.
- Sincronização de fila offline após retomada de conexão.
- Consentimento parental remoto.
- Conteúdo Kids remoto com pedido e decisão de publicação.
- Notificações parentais remotas com marcação de leitura.

## Depende do backend

- Política operacional final para foto infantil.
- Contratos finais de homologação e produção com dados reais.
- Critérios de auditoria/relatório para aceite LGPD, consentimento parental e decisões Kids.

## Depende jurídico/LGPD

- Texto definitivo de Termo de Uso.
- Texto definitivo de Aviso de Privacidade.
- Base legal e retenção para dados infantis.
- Política de visibilidade e moderação de conteúdo infantil.

## Próximo ciclo

- Validar app em dispositivo físico.
- Revisar textos de interface com acentuação e clareza.
- Consolidar critérios de homologação no `CHECKLIST.md`.
- Preparar tag/release interna `v0.1.0`.
- Definir script ou procedimento de build de distribuição.
